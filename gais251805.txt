ITEM GOING OUT
Requisition and Issue Slip (RIS)

Path A – Small / Office Supply Replenishment
Guest submits RIS for small quantities already in stock.
Mandatory fields: Department, Date, Requested By, Unit, Item Description, Qty.
Request goes to Admin for approval.
Once approved, Warehouse issues items and deducts from inventory.
This ensures every small request is validated before stock moves.

Path B – Bulk / Delivery Integration
Bulk deliveries (e.g., pallets of paper, boxes of pens) are tagged COMPLETE by Warehouse via Delivery Receipt (DR).
Instead of requiring the user to input each line item manually, the system auto-generates a bulk RIS record linked to the DR.
Admin approval is implicit because the PR for bulk items was already approved before delivery.
Warehouse can then issue items directly from the DR record, deducting stock in one action.
This avoids double entry and keeps Admin oversight intact

Administrative Approval (Admin)
The Admin reviews the submitted GSOID, he needs to fills up the PR, BUDGET, BAC and approved. The system now allows the printing of the Official PR and RIS Document.    

Delivery & Receiving (Warehouse)
Upon the items' physical arrival at the facility, Warehouse Staff retrieves GSOID, performs a technical inspection to tag the order as COMPLETE or DISCREPANCY, input value of ACTUAL ITEMS RECEIVED, input REMARKS, then official Delivery Receipt (DR). A hard copy is printed for the Supplier.

Inventory Integration (Automatic)
Once the DR is finalized, the system automatically maps and uploads item details to the INVENTORY_MASTER, this triggers a real-time update of Stock Levels across the dashboard.

Issuance & Stock Deduction (User Pickup)
The user presents their GSOID then the system automatically deducts the quantities from the inventory database.
